🔑 GitHub Access

This panel is hosted on GitHub Pages. Every "Save" here commits settings.json straight to your repo using the GitHub API. Create a Fine-grained Personal Access Token at github.com/settings/tokens — grant it Contents: Read and write for the pos repo only. The token is stored on this device only (localStorage) and sent exclusively to api.github.com.

⚠️ No token set — saves will fail

🏪 Store Status

Turning this off immediately blocks ordering on this device/site — use it for a sudden closure (staff shortage, running out of stock, unplanned holiday). This does not touch STORE_CONFIG.closedForHoliday in the code, which is for planned closures set in advance.

📣 Announcement Banner

The dismissible banner customers see at the top of the ordering page. Changing the text makes it reappear even for people who dismissed an earlier version.

Gently pulses to grab attention

🏷️ Coupon Codes

Turn any code off without deleting it, change its discount, or add a brand-new one. Turned-off codes are rejected at checkout as if they didn't exist.

CodeTypeValueLabel shown to customerActive

🍽️ Menu Control

Toggle items on/off and pick which days each one is available. Items turned off vanish from the customer menu instantly. Tick specific days to restrict an item to those days only — leave all days unticked for “every day, no restriction”. Items marked off by default in the data file are shown here too, so you can enable them when ready.

🧩 Menu Manager

Add items to an existing category, create a brand-new category, delete items you no longer want, or mark your staff favourites — all reflected on the Order Terminal instantly after saving.

Leave blank to auto-fill from category name
